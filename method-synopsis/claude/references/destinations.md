Configured defaults for where method synopses are saved. Claude reads this file on each run (Step 2).

## Per-destination notes

- **obsidian** — Must be an absolute path to a folder that already exists (or the user confirms creating it).
- **local** — Default root: `~/method-synopses/` (Windows: `%USERPROFILE%\method-synopses\`).
- **project** — Relative to workspace root; good for team repos.
- **method_playbook_folder** — Where **method-synopsis** reproducibility mode saves rerunnable study docs. See [method-playbook.md](method-playbook.md).
- **chat_only** — No file write; full markdown in the assistant message only.

## Per-run overrides

Triggers like `method synopsis → clipboard`, `method synopsis this research → project`, or `→ method-playbooks` override defaults for one run only. Do not edit this file for overrides.

## Claude-specific notes

- **Claude Code:** Claude can edit this file directly after the first-run wizard.
- **Claude Desktop / CoWork:** If file edit is unavailable, store the user's chosen path in session context and confirm each run until they edit this file manually.
